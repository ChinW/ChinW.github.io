ssh登录主机

11月 22, 2015 in ssh

今天想给自己新弄的一个主机配一下SSH，突然发现忘了部分操作细节，在此记录一下。

1.生成密钥对

两种方法生成密钥对

使用OpenSSL

生成RSA私钥

genrsa -out rsa_private_key.pem 2048

（可选）使用
pkcs8 -topk8 -inform PEM -in rsa_private_key.pem -outform PEM –nocrypt
将RSA密钥转换成PKCS8格式，密码可以为空

生成RSA公钥

rsa -in rsa_private_key.pem -pubout -out rsa_public.pem

使用ssh-gen

ssh-keygen -t [rsa|dsa]

过程当中会要求输入密码，但是如果输入了，那么远程登录主机的时候就会要求键入改密码
;如果想要的目的是免密码登录,密码请留空

2.将公钥拷贝到目标主机

将公钥拷贝到~/.ssh中，远程上传可以使用scp命令

scp -r your/target/file/path user@ip:your/remote/file/path/

如果主机下已经有authorized_keys，注意不要覆盖改文件，没有的话主动创建文件

touch ~/.ssh.authorized_keys

然后执行

cat rsa_public_key.pem >> authorized.keys

或者本地执行

cat ~/.ssh/japan_rsa.pub | ssh root@45.32.248.65 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

3.在本机添加标识

打开~/.ssh/config，没有的话就创建一个，书写格式为（替换[]中的内容为自己的配置）

4.登录

使用 ssh myHost 进行登录

5.可能出现的问题

  1. 如果出现出现
    Permissions 0644 for 'your/target/file' are too open.
    把权限改成0600就好了
  2. OpenSSL 生成的密钥会出现key_load_public: invalid format导致无法登录.
    OpenSSL生成的公钥和ssh-keygen生成的公钥的格式不一致，ssh-keygen生成公钥的格式是OK的。
